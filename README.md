# 会议记录小程序（Meeting Minutes MiniApp）

一个用于**会议实时记录与会后自动整理**的小程序方案：

- 会议中：实时语音转写（ASR）
- 会议后：自动按说话人、议题、结论、待办整理纪要
- 场景化：例如需求评审会，自动提取“需求点 / 风险 / 待确认项”

---

## 1. 目标与核心能力

### 1.1 目标

让用户在开会时只需打开小程序并点击“开始会议”，即可在会后获得可直接转发的结构化会议纪要。

### 1.2 核心能力（MVP）

1. **实时录音 + 流式转写**
2. **说话人识别（Speaker Diarization）**
3. **自动生成纪要**
4. **纪要导出/分享**

---

## 2. 建议技术方案

### 2.1 小程序端（微信）

- 页面：
  - `pages/index`：创建/进入会议
  - `pages/meeting`：录音中 + 实时转写展示
  - `pages/summary`：纪要预览与导出
- 录音：微信小程序录音 API（`RecorderManager`）
- 上传：分片上传音频到后端（避免大文件失败）

### 2.2 后端（云函数 / Node.js 服务）

- 接收音频流或音频文件
- 调用 ASR 服务（推荐阿里云通义听悟）
- 保存原始转写、说话人标签、时间戳
- 调用大模型生成结构化纪要

### 2.3 AI 能力

- **语音识别（ASR）**：通义听悟（支持说话人分离、时间戳）
- **纪要总结（LLM）**：通义千问 / 其他兼容模型
- 输出 JSON 结构，前端再渲染成可读页面

```json
{
  "speaker": "speaker_1",
  "startMs": 12340,
  "endMs": 15620,
  "text": "这个需求我们需要在五月底上线。"
}
```

## 4. 纪要生成提示词（可直接用）

你是资深项目经理，请根据会议转写内容输出结构化纪要，要求：

1. 按“议题”分组。
2. 每个议题输出：背景、结论、待办（责任人+截止日期）、风险。
3. 如果无法确定责任人或日期，标记为“待确认”。
4. 输出严格遵循 JSON，不要输出多余文字。

输入：`{{transcript_text}}`
